装饰器模式：允许向一个现有的对象添加新的功能,同时又不改变其结构。它是现有类的一个包装

装饰器模式是适配器模式的一种特殊表现形式。

    装饰器模式                                  适配器模式
是一种非常特别的适配器模式                   可以不保留层级关系
装饰者和被装饰者都要实现同一个接口           适配者和被适配者没有必然的层级联系
主要目的是为了继承类别。                     通常采用代理或者继承形式进行包装
满足is-a的关系                               满足has-a
注重的是覆盖、扩展                           注重兼容、转换

spring中有decorator或者wrapper的类都是装饰器模式

代码例子：有一个果汁店，需要计算混合果汁的价格，这时候可以用装饰器模式，一层一层计算价格
