单例模式：
一个类只能有一个实现，并且只能对外提供这一个实例

分为饿汉式、懒汉式、注册登录式、序列化式

饿汉式的，初始化就加载，线程安全，耗内存，不管用没用都占用

  例子：家里只有一台中央空调，不管天气热不热只要有电就开着很耗电

懒汉式单例，延迟加载，等真正需要用到的时候再实例化

   优点：节约资源，但是存在线程安全问题，加锁影响性能，用静态内部类实现
   即解决了延迟加载，又解决了不加锁的线程安全，性能高，推荐使用

   例子：家里唯一的中央空调，不热的时候不开，等热的时候再开

注册登录式spring中用的比较多，属于单例的集合版，同一个class只能有一个单例，单例类维护许多class的单例集合
