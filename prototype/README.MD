原型模式：
把对象中配置的依赖关系，在每次使用对象前都会创建一个新对象，并且将依赖关系完整地赋值给这个新建的对象，也就是深度克隆。原型模式的实现方式有很多种，例如反射机制（apache，spring用此种方式），序列化和反序列化，还有object的clone方法，本例使用clone方法

使用场景：创建对象如果成本很高，通过new对象很浪费资源，这时候就可以用原型模式

原型模式有很多种应用，例如DTO中的值会赋值传递给VO，VO再赋值传递到View中去
生活中的场景：高仿名牌衣服，模仿名牌衣服的样式，制作假的高仿衣服，因为真的衣服成本很高，所以用clone的方式降低成本。

clone方法会深度克隆8种基本类型，对于引用类型，只复制值，属于浅度克隆，所以对于所有的引用类型都需要再次调用clone方法，得到的新对象设置到原引用类型中。
优点：性能高
缺点：引用类型很多，需要clone很多次，实现麻烦
